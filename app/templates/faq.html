Perguntas Frequentes

Entenda como a plataforma funciona.

Como funciona a análise?
A análise é realizada em múltiplas etapas:
  • Análise Multi-Fonte: Consultamos APIs como VirusTotal e Google Safe Browsing simultaneamente.
  • Análise Estática Local: Verificamos o arquivo localmente (hash SHA256, tipo de arquivo) antes de enviar dados externos.
  • Inteligência Artificial: Utilizamos modelos avançados (Gemini, Groq) para interpretar os dados técnicos e gerar um resumo executivo em linguagem natural.
O que é a Detecção de Phishing Avançada?
O sistema identifica URLs que utilizam caracteres especiais (como cirílicos) para se passar por domínios legítimos. Essa técnica, conhecida como ataque de homógrafo, é comum em tentativas de phishing e é detectada automaticamente pela nossa ferramenta.
O que significa a pontuação de risco?
A pontuação de risco é um cálculo consolidado que combina os vereditos de todas as ferramentas de segurança consultadas. Ela varia de 0 a 100, onde valores mais altos indicam uma maior probabilidade de o arquivo ou URL ser malicioso.
Guia Passo-a-Passo: Como obter cada API Key

Siga estas instruções para obter as chaves necessárias gratuitamente:

1. VirusTotal (Análise de Malware)
  • Cadastre-se em VirusTotal Join Us.
  • Após logar, clique no seu perfil (canto superior direito) e vá em "API Key".
  • Copie a chave exibida em "Personal API Key".
2. Groq (Inteligência Artificial - Recomendado)
  • Acesse Groq Console.
  • Faça login (pode usar conta Google).
  • Clique em "Create API Key", dê um nome e copie a chave gerada.
3. Google Gemini (IA Alternativa)
  • Acesse Google AI Studio.
  • Clique em "Create API key".
  • Escolha o projeto e copie sua chave.
4. Google Safe Browsing (Verificação de URLs)
  • Acesse o Google Cloud Console.
  • Crie um novo projeto ou selecione um existente.
  • Vá em "APIs & Services" > "Library" e pesquise por "Safe Browsing API". Ative-a.
  • Vá em "Credentials" > "Create Credentials" > "API Key".
5. Elastic Stack & Wazuh (SIEM Locais Gratuito)
  • Elastic: Rode docker-compose up -d. O Elasticsearch ficará em http://localhost:9200 e o Kibana em http://localhost:5601.
  • Wazuh: Incluído no nosso docker-compose.yml! O indexador roda na porta 9201, o Manager na 55000 e o Dashboard na 5602.
  • Dica: Para o Wazuh funcionar no Windows (Docker Desktop), garanta que o WSL2 tenha memória suficiente (mínimo 4GB recomendados).
Onde configurar as chaves?
Após obter suas chaves, clique no ícone de engrenagem no menu lateral ou acesse a página de Configurações. Cole as chaves nos campos correspondentes e clique em "Salvar Alterações". A aplicação estará pronta para uso imediato.
Integração com SIEM/SOAR (Elastic & Wazuh Locais)

O The APEX suporta integração com o Elastic Stack e Wazuh, que podem ser utilizados gratuitamente se instalados localmente (Self-Managed).

  • Setup Automático: Incluímos um docker-compose.yml na raiz do projeto que configura o Elasticsearch, Kibana, Wazuh Indexer, Manager e Dashboard automaticamente.
  • Requisito Wazuh: No Linux ou WSL2, você precisa rodar o comando sudo sysctl -w vm.max_map_count=262144 antes de iniciar para que o indexador funcione corretamente.
  • Configuração: Use http://localhost:9200 para o Elastic e http://localhost:55000 para o Wazuh nas Configurações.

Nota: Mantemos a remoção de Splunk, QRadar e Microsoft Sentinel por serem serviços predominantemente pagos ou corporativos.
